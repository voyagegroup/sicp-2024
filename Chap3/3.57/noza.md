# 問題 3.57

`add-streams` を使った `fibs` を、`delay` がメモ化をしない単なる `(lambda () ⟨exp⟩)` だとすると、同じストリーム要素を得るたびに遅延計算が再実行され、通常の再帰版 `fib` と同じ計算木が展開される。

add-streams 版の `fibs` で `n` 番目 (0 始まり) の Fibonacci 数を得るときの加算回数を `A(n)` とすると、

- `A(0) = 0`, `A(1) = 0`（基底は加算なし）
- `A(n) = A(n-1) + A(n-2) + 1` （`n>=2`。この階層で 1 回加算）

この漸化式の解は `A(n) = F_{n+1} - 1`（`F_k` は `F_0=0, F_1=1` の Fibonacci 数）なので、n 番目の Fibonacci 数を計算するには加算がちょうど `F_{n+1}-1` 回実行される。`F_{n+1} ≈ (φ^n)/√5`（`φ` は黄金比）ゆえ、加算回数は指数的に増大する。
