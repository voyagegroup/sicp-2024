```scheme
(define x 10)

(parallel-execute (lambda () (set! x (* x x)))
                  (lambda () (set! x (* x x x))))
```

* 1,000,000
  * p1がxを100に設定し、次にP2がxを1,000,000に設定する
  * p2がxを1000に設定し、次にp1がxを1,000,000に設定する
* 10,000
  * p1がxに2度目のアクセスをした時にp2がxを1,000に設定する
    * 10*1,000
  * p2がxに3度目のアクセスをした時にp1がxを100に設定する
    * 10*10*100
* 100,000
  * p2がxに2度目のアクセスをした時にp1がxを100に設定する
    * 10*100*100


```scheme
(define x 10)

(define s (make-serializer))

(parallel-execute (s (lambda () (set! x (* x x))))
                  (s (lambda () (set! x (* x x x)))))
```

直列に行われるため、p1が終わった後にp2が行われるかその逆が起きうる可能性である。

* 1,000,000
  * p1がxを100に設定し、次にP2がxを1,000,000に設定する
  * p2がxを1000に設定し、次にp1がxを1,000,000に設定する
